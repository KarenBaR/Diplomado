Qué estamos midiendo

El incremental no es la variación mes a mes, sino el aporte atribuible a haber priorizado estos clientes como P1. La pregunta de fondo es: si estos mismos leads se hubieran trabajado a una efectividad “normal, sin priorización” (la base), ¿cuántos desembolsos y cuántos soles se habrían colocado? Todo lo que P1 coloca por encima de ese contrafactual es el impacto del modelo.

Aquí fijamos la efectividad base en 0.6% (cercana al rendimiento del siguiente nivel de prioridad, P2). El resultado es sensible a esta base: subirla reduce el incremental; bajarla lo amplía.

Cómo se calcula

1

Efectividad real

Qué tan bien convierte P1 lo que gestiona.
desembolsos / gestionado
2

Desembolsos base

Lo que daría la efectividad “sin modelo”.
gestionado × 0.6%
3

Desembolsos incrementales

El extra atribuible a P1.
reales − base
4

Monto incremental

Valorizado al ticket P1 del mes.
desemb. incr. × ticket
efectividad_real = desembolsos / gestionado
desembolsos_base = gestionado × 0.006
desembolsos_incr = desembolsos − desembolsos_base
ticket_p1 = monto_desembolsado / desembolsos
monto_incremental = desembolsos_incr × ticket_p1

Cómo sacarlo en SQL (Athena)

La base de efectividad está parametrizada en un CTE — cambia el 0.006 para probar otros escenarios.

-- Impacto incremental P1 · base efectividad = 0.6% · canal TLV B / Capital de Trabajo
WITH params AS (
    SELECT CAST(0.006 AS DOUBLE) AS efectividad_base
),
base AS (
    SELECT
        PERIODO,
        SUM(NRO_LEADS)        AS leads,
        SUM(FLG_GESTIONADO)   AS gestionado,
        SUM(FLG_DESEMBOLSADO) AS desembolsos,
        SUM(COLOCACION)       AS monto
    FROM T_BN_FUNNEL_CANALESCOM_CLIENTE_HISTORICO
    WHERE barr_variable_1   = 'TLV B'
      AND ID_CANAL          = 3
      AND BARR_COD_HORARIO  = 'CAPITAL DE TRABAJO'
      AND PRIORIDAD_GRUPO_FINAL = 'P1'
    GROUP BY PERIODO
)
SELECT
    b.PERIODO,
    b.gestionado,
    b.desembolsos,
    b.monto,
    ROUND(CAST(b.desembolsos AS DOUBLE) / NULLIF(b.gestionado,0) * 100, 2)  AS efectividad_real_pct,
    ROUND(b.gestionado * p.efectividad_base, 1)                            AS desembolsos_base,
    ROUND(b.desembolsos - b.gestionado * p.efectividad_base, 1)            AS desembolsos_incr,
    ROUND(b.monto / NULLIF(b.desembolsos,0), 0)                          AS ticket_p1,
    ROUND( (b.desembolsos - b.gestionado * p.efectividad_base)
           * (b.monto / NULLIF(b.desembolsos,0)) , 0)                     AS monto_incremental
FROM base b
CROSS JOIN params p
ORDER BY b.PERIODO;
ℹ
Para el total acumulado, envuelve la consulta y haz SUM(monto_incremental). El último periodo puede estar parcial (mes en curso); en el seguimiento se excluye para no distorsionar la tendencia.
